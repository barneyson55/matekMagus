# CI-001 â€” GitHub Actions CI for lint + tests

## Goal
Add a CI pipeline that verifies every PR:
- npm ci
- formatting check
- unit tests
- e2e smoke tests (at least on ubuntu-latest with xvfb strategy)

## Constraints
- Keep it fast and stable.
- Do not build release installers in CI yet (separate pipeline later).
- Use Node version aligned with project (prefer existing engines/.nvmrc; else Node 18).

## Implementation (when autopilot executes this task)
1) Create .github/workflows/ci.yml:
   - on: pull_request, push
   - jobs:
     - test (matrix: ubuntu-latest, windows-latest)
       - run unit tests on both
       - run e2e on ubuntu-latest (headless)
       - optionally skip e2e on windows if flakey at first
2) Add caching for npm (actions/setup-node cache: 'npm').
3) Ensure CI uses `npm ci`.

## Definition of Done
- CI runs on PR and green on main branch.
- Failures show actionable logs.
